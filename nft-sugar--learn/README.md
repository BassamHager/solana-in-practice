# nft-sugar-dapp

A minimal TypeScript project that demonstrates how to create a **fungible SPL token on Solana** using the Metaplex Foundation Token Metadata Standard + QuickNode as RPC provider.

Based on the QuickNode guide ["Create a Solana SPL Token with Metaplex"](https://www.quicknode.com/guides/solana-development/spl-tokens/how-to-create-a-fungible-spl-token-with-the-new-metaplex-token-standard) â€” modified for a local project setup.

## âœ… What this does

- Creates a new Solana wallet (devnet) and airdrops SOL for testing.
- Uploads a token icon and metadata JSON to IPFS (or you can use any public URL).
- Mints a new fungible SPL token with metadata (name, symbol, URI) according to the Metaplex standard.
- Prints the token mint address and confirms the transaction so you can view the token on Solana explorers or add it to a wallet (e.g. Phantom).

## ğŸš€ Setup & Usage

1. Clone this repository:
   ```bash
   git clone <your-repo-url>
   cd <your-repo>
   ```

2. Install dependencies:
   ```bash
   yarn install
   # or
   npm install
   ```

3. Create a keypair and request airdrop (devnet):
    ```bash
    ts-node wallet.ts
    ```
    This will generate guideSecret.json and log your public address.

4. Upload your token image (e.g. mytoken.png) and metadata JSON (e.g. token.json) to IPFS (or another public storage). The metadata JSON should follow the standard format:
    ```json
      {
        "name": "My Token Name",
        "symbol": "MTK",
        "description": "My custom token on Solana",
        "image": "IPFS_OR_PUBLIC_URL_TO_IMAGE"
      }
    ```
    this allows wallets/explorers to display token info correctly. Quicknode +1

5. Edit mint.ts, set your RPC endpoint URL and the metadata URI, then run:
   ```bash
      ts-node mint.ts
   ```
   On success you should see output like:
Successfully minted X tokens ( <mint-public-key> )

6. Use the printed mint address to view your token on a Solana explorer or add it to your wallet to check the balance.

## âš ï¸ Security / Important Notes

- Never commit your private key or guideSecret.json to a public repository. Use environment variables or .gitignore to keep secrets safe.

- The metadata (especially image + JSON) should be publicly accessible (e.g. pinned on IPFS or stored on public CDN) so wallets/explorers can fetch and display it. Quicknode +1

- This guide uses devnet. For mainnet usage, make sure to understand fees, minting authorities, and implications carefully.
